## DB

- 선택정렬: 최소값을 찾아 첫 원소와 교환 => 교환한 원소를 제외하고 반복
- Cardinality: 튜플의 갯수 => 레코드 갯수
- Degree: 속성의 수 ==> 필드 갯수
- 전위순회: 방문 => 왼쪽 방문 => 오른쪽 방문
- 중위순회: 왼쪽 방문 => 방문 => 오른쪽 방문
- 후위순회: 왼쪽방문 => 오른쪽 방문 => 방문



- DB 정규화
- 1NF: 컬럼이 원자값을 가짐
- 2NF: 함수적 종속 제거
- 3NF: 이행 종속 제거
- BCNF: 결정자이면서 후보키가 아닌 함수 종속 제거
- 5NF: 조인 종속성



- 스키마 종류: 외부 - 개념 - 내부(물리)
- DB 설계: 개념(사용자 요구) => 논리(테이블 설계) => 물리 설계
- 물리적 설계: 파일저장 구조, 엑세스 경로 결정
- 논리적 설계: 스키마 설계



- 선형 탐색 평균 횟수: `sum(n) / n` => `(n+1)/2`
- 뷰: 논리적 독립성 가짐, 삽입, 삭제, 갱신시 제약조건 있음
- 타임스탬프 기법: 동시성 제어를 위한 직렬화 기법, 트랜잭션간 순서를 미리 정함
- 트리의 차수: 자식의 갯수
- 해시 테이블의 오버플로우 처리: 개방주소, 폐쇄주소, 재해싱



- 무결성 제약 조건: 개체(NOT NULL, 중복 X), 참조(외래키), 도메인
- 색인 순차 파일 구조: index, Prime, Overflow
- 수퍼키: 유일성
- 후보키: 유일성, 최소성
- 기본키: 선택된 후보키 1개
- 대체키: 기본키가 아닌 후보키
- 로킹 단위가 클때: 오버헤드(로킹 갯수) 감소, 공유도 감소
- 관계 대수: 절차적 언어, 릴레이션 조작을 위한 연산집합, 연산순서 명시
- ORDER BY - HAVING

## 전자계산기

- PE 연산기: Vector Processor
- 1K = 2^10^
- DMA 구성요소: 인터페이스 회로, 레지스터(주소, 워드 카운트, 제어, 데이터), 주소라인
- 외부 인터럽트: 입출력, 타이머(신호), 정전
- 내부 인터럽트: Trap, 잘못된 명령
- SW 인터럽트: 다른 프로그램 실행
- 0주소: 스택          1주소: 누산기            2주소: 레지스터
- 주소 레지스터 비트: 가상 주소 공간 비트 + 기억공간 비트
- 채널: CPU의 개입 없이 입출력 완료
- MBR: 기억장치에 출입하는 데이터의 버퍼
- MAR: 기억장치에 출입하는 데이터의 주소를 기억
- 나노 초(ns): 10^-9^초
- 1024 = 2^10^
- 0 주소 명령: 스택
- 1 주소 명령: 누산기(Accumulator)
- 상태 레지스터: PSW가 저장되는 레지스터
- 인덱스 레지스터: 반복 횟수, 서브루틴 연결 수행
- 인코더: 2^n^ ==> n
- 디코더: n ==> 2^n^
- 멀티플렉서: n => 1
- 디멀티플렉서: 1 => n
- 3-초과코드(Excess-3): `origin + 3`한 값으로 표현
- 마샬링: 한 객체의 메모리에서의 표현방식을 저장 or 전송에 적합한 다른 형식으로 변환
- 주소 지정 방식에 따른 연산 속도: 즉시(immediate) < 직접(direct)(참조 1회) < 간접(indirect) (참조 2회)
- PLA: Programmable Logic Array, n개 입력 ==> 최대 2n개 출력
- DMA: 버스사용요구->허가->전송->인터럽트 (전송 후 알림)



캐시 기억 장치 매핑 방법

- 연관 매핑: 워드, 번지, 데이터를 함께 저장. 빠름, 복잡하지만 낮은 미스율
- 세트-연관 매핑: 직접 + 연관, 개시를 세트로 분할함
- 직접 매핑: 인덱스 필드로 캐시 접근에 사용. 간단한 매핑, 높은 미스율



- CPU의 4단계: Fetch, Indirect, Execute, Interrupt
- RS: 0, 1저장 or 현상태
- D: 입력 그대로
- M/S: 출력물의 일부가 입력에 피드백 (Master-Slave)
- T: 0-현상유지, T-보수
- 제어장치 입력: 명령어 레지스터, 클록, 플래그
- 2의 보수: 반전 후 `+1`
- 프로세서의 성능지표: MIPS (Million Instructions Per Second)
- 누산기: 연산결과를 임시저장



문자 데이터의 표현

- BCD코드: 존2 + 숫자4
- ASCII: 존3 + 숫자4
- EBICIC: 존4 + 숫자4
- UNICODE: 16



채널 종류

- 셀렉터: 1개의 고속 장치와 입출력
- 블록: 여러개의 고속장치와 입출력
- 멀티플렉서: 여러 저속 장치를 제어

## 운영체제

- 파일 디스크립터: 파일시스템이 관리, 직접 참조 불가능, 파일 실행시 메모리로 이동
- 스케줄링 방식 - 선점형: RR, SRT, MFQ
- 스케줄링 방식 - 비선점형: HRN(Highes Response Ratio Time), FCFS, SJF
- 시간지역성: 하나의 저장소가 가까운 미래에 다시 참조될 확률이 높음
- 공간지역성: 일정 공간을 집중해서 참조
- 파일작성자: 수정시간, 파일명, 접근 횟수, 구조, 생성일, 접근제어 저장
- 로더의 기능: 할당, 재할당, 링킹, 로딩
- 링커의 기능: 목적프로그램을 실행가능한 로드 모듈로 변환
- Working Set: 일정시간동안 자주 참조하는 페이지들의 집합
- Fragmentation: 외부단편, 내부단편



- 디렉토리 구조
- 단일 구조: 모든파일이 같은 디렉토리
- 2단계 구조: 마스터, 유저 디렉토리로 구성, 유저 디렉토리의 자식은 파일
- 트리 구조: 2단계 구조를 확장, 하나의 루트, 다수의 종속 디렉토리로 구성
- 비주기 구조: 서브 디렉토리를 공유할 수 있도록 허용, 비주기 그래프로 구성



- 매크로 콜: 매크로에서 정의된 매크로 이름을 주프로그램에 기술
- 교착상태 해결방법 - 회피: 교착상태가 발생하면 적절히 **회피**, 은행원 알고리즘
- 분산형 OS에서 Site간 마이그레이션: Data, Computation, Process
- OS 발달과정: 일괄- 시분할 - 다중 모드 - 분산처리
- 프로세스가 전송하는 메시지 형태: 형식, 가변길이, 고정길이
- 매크로 프로세서의 기본기능: 정의인식 및 저장, 호출인식 및 확장
- 자원보호기법: 접근 제어 행렬 and 리스트, 전역 테이블, 권한 리스트
- 교착상태는 불안전상태
- 불안전상태: 특정 순서로 자원을 할당해, 결국 교착상태가 발생할 수 있는 상태
- 안전상태: 작업을 완료할 수 있는 상태

## 소프트웨어 공학

* 소프트웨어 재공학: 구조적 재설계

* 예방 보수: 유지보수성, 신뢰성, 재사용성 개선 (Preventive Maintenance)

* 완전화 보수: 새로운 기능 추가, 성능 개선 (Perfective Maintenance)

* 수정 보수: 하자 보수 (Corrective Maintenance)

* 적응 보수: 환경 적응을 위한 보수 (Adaptive Maintenance)

* 결합도: 자료 < 스탬프 < 제어 < 외부 < 공통 < 내용 (결합도가 낮을 수록 높은 독립성)

* 블랙박스 테스트: 동치분할, 경계값 분석, 비교, 원인 효과

* 화이트박스 테스트: 데이터 흐름, 루프, 조건, 기초경로

* 일정 계획 기법: 작업 분해, PERT/CPM, 간트차트

* 럼바우 분석 기법: 객체(객체 특성 규명) -> 동적(객체간 흐름제어) -> 기능(자료흐름도 기반 처리과정) 모델링

  ​		==> 기능: 입출력 결정->자료흐름도작성->기능서술->제약사항결정

* Putnam 모형: SW 개발주기 간 요구인력 분포 모형 (SLIM 도구 사용)

* COCOMO 유형: organic, semi-detach, embedded 클래스로 구성, LOC

## 데이터 통신

* OSI: 물리 -> DL -> 네트워크 -> 전송 -> 세션 -> 표현 -> 응용
* IPv4: Broadcast, Multicast, Unicast
* IPv6: Anycast, Multicast, Unicast
* IPv4 => IPv6 전환 전략: Dual Stack, Tunneling, Header Translation
* 피기배킹 응답: Information 프레임
* Framing Error: 송/수신측의 샘플링 시점이 달라 발생하는 오류, 비동기식 전송에서 발생
* ASK: 진폭 변조
* PSK: 위상 변조
* QAM = ASK + PSK
* DCE - DTE간 접속 규격: X 시리즈 (비동기: X.20, 동기: X.21, 패킷: X.25)
* 패킷교환: 패킷 단위로 **저장**, 전달
* 회선교환: 고정된 회선으로 전달, 연결 이후 사용자에 의해 오류제어
* 4 위상 변조: `00`, `01`, `10`, `11` ==> 2비트
* VAN: 회선을 임차, 부가가치성 데이터 서비스 제공
* Packet Assembler/Disassembler: 일반형 터미널에서 패킷 조립, 분해 기능 수행
* 패킷 교환망의 경로배정방법: 고정경로(static), 범람 경로(flooding), 적응 경로(adaptive), 임의 경로(random)
* 통신용량: 대역폭 * `log2 (1 + (신호대 잡음비))`
* 802.2 LLC
* 802.3: CSMA/CD      802.4: 토큰버스      802.5: 토큰링
* HDLC 전송 제어 절차: NRM, ARM, ABM
* Polling: 단말에서 제어국 방향으로 데이터 전송 (멀티포인트 네트워크)
* 원천 부호화: 데이터 크기를 줄이기위해 압축, DPCM, DM, LPC
* TCP/IP 구조: 응용, TCP, IP, 링크계층
* 무선랜에서 경쟁에 의한 접근제어: DCF, (PCF는 무경쟁)
* 블루투스의 물리계층: RF (TTD: 블루투스의 양방향 통신 방식)
* 맨체스터 코딩: 전압의 승압, 감압으로 비트 표현
* IP: 비연결형, 오류검사 X
* 통신프로토콜의 기본요소: 구문, 의미, 타이밍
* 거리벡터 라우팅 프로토콜: RIP
* QPSK: 4위상 편이변조(한 위상-2bit)
* DPSK: 2위상 편이 변조
* ODPSK: 8위상 편이변조
* 종단 시스템의 응용프로그램 간 대화 관리: 세션계층
* Go-Back-N ARQ: 오류발생시, 오류발생 프레임 + 직전 프레임
* 통계적 시분할: 전송할 데이터가 있는 터미널에만 시간슬롯 할당
* RIP: 거리 벡터 라우팅 프로토콜, 최대 15홉, 벨만포드, 소규모
